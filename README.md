# Elastic Stack Deployment

Three-node Elasticsearch 8.12.0 cluster deployed with Ansible and Docker Compose, with TLS encryption, X-Pack security, Kibana for visualization, Metricbeat for cluster monitoring, and Filebeat for log shipping (SIEM-style).

## Project Structure

│   ├── inventory.ini               # Inventory (localhost)

│       │   ├── tasks/main.yml

## Requirements

- Docker + Docker Compose v2
- Ansible with `community.docker` collection
- ansible-vault (for secrets)

Install the required Ansible collection:
```bash
ansible-galaxy collection install community.docker
```

## Configuration

Cluster parameters are defined in `ansible/roles/elasticsearch/vars/main.yml`:

| Variable       | Default              | Description           |
|----------------|----------------------|-----------------------|
| `es_version`   | `8.12.0`             | Elasticsearch version |
| `cluster_name` | `ml-research-cluster`| Cluster name           |
| `es_java_opts` | `-Xms512m -Xmx512m`  | JVM heap size          |
| `es_nodes`     | 3 nodes              | Node names and ports   |

### Services

| Service     | Port | Purpose                            |
|-------------|------|-------------------------------------|
| es-node01   | 9200 | Elasticsearch node                  |
| es-node02   | 9201 | Elasticsearch node                  |
| es-node03   | 9202 | Elasticsearch node                  |
| Kibana      | 5601 | Web UI, search, dashboards          |
| Metricbeat  | —    | Cluster health/resource monitoring  |
| Filebeat    | —    | Log shipping for security events    |

## Secrets

The `secrets.yml` file is encrypted with ansible-vault and contains:
- `es_password` — Elasticsearch `elastic` user password

To edit secrets:
```bash
ansible-vault edit ansible/roles/elasticsearch/vars/secrets.yml
```

## Deployment

```bash
ansible-playbook ansible/deploy_cluster.yml --ask-vault-pass --ask-become-pass
```

The playbook will:
1. Generate TLS certificates via `elasticsearch-certutil` (skipped if already exist)
2. Set `vm.max_map_count=262144` on the host
3. Render per-service `docker-compose-*.yml` files from Jinja2 templates
4. Start Elasticsearch, wait for it to become healthy
5. Start Kibana, Metricbeat, and Filebeat
6. Wait for the cluster to reach `green` or `yellow` status

## Verify the Cluster

Check cluster health:
```bash
curl -k -u elastic:<password> https://localhost:9200/_cluster/health?pretty
```

Check all nodes are joined:
```bash
curl -k -u elastic:<password> https://localhost:9200/_cat/nodes?v
```

Verify memory lock is active:
```bash
curl -k -u elastic:<password> https://localhost:9200/_nodes?filter_path=**.mlockall
```

Open Kibana:
```
http://localhost:5601
```
Login with `elastic` / your password.

## Monitoring

Metricbeat collects node-level metrics (JVM heap, disk, shard count) and ships them to Elasticsearch, visible under **Stack Monitoring** in Kibana.

## Log Shipping / SIEM

Filebeat ships logs to Elasticsearch for security-relevant analysis (authentication attempts, system events). Indices appear under the `filebeat-*` pattern and can be explored in Kibana Discover or built into Security/SIEM dashboards.

## Security

- TLS enabled for both HTTP and transport layers
- Certificates generated with `elasticsearch-certutil`
- Secrets stored encrypted via ansible-vault
- `certs/`, `*.p12`, `*.key`, `*.pem`, and all generated `docker-compose*.yml` / service config files are gitignored

## Teardown

```bash
docker compose -f docker-compose-es.yml -f docker-compose-kibana.yml -f docker-compose-metricbeat.yml -f docker-compose-filebeat.yml down -v
```
> `-v` removes named volumes (all indexed data will be lost).
